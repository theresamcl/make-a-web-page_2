Intro to Programming Class Notes

How the Web Works

The web consists of people and computers communicating with each other via 3 main components:

  • Browser - computer/human interface for the web; i.e., Mozilla, Internet Explorer, Safari, Chrome
  • Internet - the world's largest computer network
  • Servers - host files with content including text, audio, images and video

HTML

HTML stands for Hypertext Markup Language. HTML documents form the majority of the content on the web. HTML documents contain text content which describes "what you see" and markup which describes "how it looks". For more information about current HTML standards check out the w3.org page.

Tags and Elements

HTML elements consist of HTML tags + content. The tags contain formatting information telling the browser how the associated content should appear and function; i.e., bold, italicized, simple content or a link to another webpage. Most HTML tags have the format <opening tag> content </closing tag> although some, like the image source tag, are designed without a closing tag.

Inline vs Block Elements

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them, imparting additional formating attributes.

Box sizing

Boxes have 4 characteristics. Working from the inside out they are:

  • Content,
  • Padding,
  • Border, and
  • Margin.
It is best to set sizes as a percentage rather than in pixels, although a maximum size may be specfied in pixels.

Box positioning

The Flex box allows divs to appear side by side on the page rather than the default of appearing on sequential separate lines. Use webkits to enable flex boxes to work properly in different browsers.

Defining Classes and Avoiding Repetition

When dividing the design mock into boxes, watch for repeated elements that require consistent formatting, such as the Title bars in these notes. Instead of writing a generic <div> container and specifying the formatting for each one, define a class that can be used to specify the formatting in the css file. Features like background color, font, margins and padding can be modified through the class definition.

Computers interpret instructions literally, and a small mistake like a missing ">" can cause unintended behavior in a program. Applying changes to the class definition instead of multiple instances reduces topportunities for typos and omissions. Modifying via the class also shortens the time needed to code new instances and changes to style elements.

CODE - TEST - REFINE

When creating a website from a design mockup, use the Code-Test-Refine stategy:

  1. Look for natural boxes on the design mockup.
  2. Look for repeated styles and semantic elements.
  3. Write your HTML.
  4. Apply styles from the biggest elements to the smallest. Use classes to describe styles for repeated elements efficiently, without repetition.
  5. Fix things. Save your file and refresh the browser to check your changes as you go. Use Chrome developer tools to model style changes quickly, then transfer the best settings to your code file.

Check your code
